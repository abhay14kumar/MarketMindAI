# Contributing to MEKS

## Add or Update Knowledge

1. Find the canonical topic in the [Master Index](./indexes/MASTER_INDEX.md).
2. Update its status and progress metadata.
3. Put detailed teaching material in `academy/`, practical work in `labs/`,
   and product truth in `docs/` or `architecture/`.
4. Add reciprocal links from the topic manifest to the new evidence.
5. Use the relevant template.
6. Review technical accuracy, security, discoverability, and link integrity.

## Evidence Labels

| Label | Meaning |
|---|---|
| `implemented` | Working repository capability |
| `tested` | Automated or documented verification exists |
| `operated` | Runtime or incident evidence exists |
| `explained` | Reviewed learning material exists |
| `interviewed` | Question set and follow-ups completed |
| `revised` | Revisited after new evidence |

## Definition of Done

- Metadata and owner are current.
- Prerequisites and related topics are linked.
- MarketMind evidence is attached.
- Security, production, and failure scenarios are considered.
- Mastery claims are supported by reviewable evidence.
- Relevant indexes or graphs are updated.
